01
Whisper — Speech to text
Extract the transcript from any video, in any language, with timestamps
Open source — OpenAI Runs locally — no API key 99 languages supported
What it does
Listens to the video audio and outputs a full transcript with word-level timestamps. Supports 99 languages. Runs on CPU or GPU.
Recommended model
medium — best balance of speed and accuracy for most hardware. Use large-v3 for production quality if you have a GPU.
Example output — transcript.txt
"In today's video we're going to look at how machine learning is changing the film industry forever. Let me show you three examples that will blow your mind..."
Use large-v3 for non-English source videos (Hindi, Arabic, etc.) — it handles accented speech much better than medium.
02
LibreTranslate — Text translation
Translate the English transcript into Spanish, Hindi, Arabic, Japanese, and more
Open source — self-hostable Free — no API key needed 30+ languages
Option A — Self-hosted
Run LibreTranslate on your own machine. Unlimited requests, fully private, no API key. Takes ~5 min to set up.
Option B — Public API
Use libretranslate.com free API. Rate-limited but fine for testing. Switch to self-hosted for client work.
Output files generated
transcript_spanish.txt → "Hola a todos, bienvenidos a este video..." transcript_hindi.txt → "नमस्ते सभी को, इस वीडियो में आपका स्वागत है..." transcript_arabic.txt → "مرحباً بالجميع، أهلاً بكم في هذا الفيديو..." transcript_japanese.txt → "みなさん、こんにちは。このビデオへようこそ..."
For better quality on complex scripts (Hindi, Arabic), consider using the Argos Translate library instead — it has dedicated models trained on those language pairs.
03
Coqui TTS — Voice cloning
Clone the original speaker's voice and generate speech in the target language
Open source — Mozilla/Coqui Zero-shot voice cloning 17 languages
How voice cloning works
You give Coqui a 6–30 second audio clip of the original speaker. It extracts their voice profile and uses it to speak the translated text.
Best model to use
XTTS-v2 — Coqui's best multilingual voice cloning model. Supports 17 languages including Spanish, Hindi, Arabic, Japanese, Portuguese.
Input required
6–30 seconds of clean audio from the original speaker. The pipeline extracts this automatically from the video.
Output produced
4 separate audio files — one per language — in the original speaker's cloned voice.
For best voice clone quality, use 15–30 seconds of clear speech with no background music. Extract from the quietest section of the original video.
04
Wav2Lip — Lip sync
Sync the speaker's lip movements in the video to the new cloned audio
Open source — L V Kumar et al. GAN-based lip rendering GPU recommended
What it does
Takes the original video (face) and the new translated audio, and re-renders the mouth/lip region to match the new speech. Output looks native.
Hardware
Works on CPU but is slow (~10min/min of video). With an NVIDIA GPU (RTX 3060+) it runs at ~real-time speed. Google Colab works too (free GPU).
Model required
wav2lip_gan.pth (~436MB) — the GAN version produces much sharper, more realistic lip renders than the standard model.
Output produced
4 lip-synced videos — one per language — where the speaker's mouth visually matches the new audio perfectly.
Use wav2lip_gan.pth (not wav2lip.pth) — the GAN version produces much sharper lip renders. Works best with a forward-facing face clearly visible in the video.
05
FFmpeg — Final render
Merge everything, add subtitles, clean audio levels, and export the final deliverable
Open source — FFmpeg team Industry standard Free forever
What it handles
Final video + audio merge, quality normalisation, subtitle burning, format conversion, compression for YouTube upload.
Output format
H.264 MP4 at 1080p, AAC audio — the YouTube-optimal format. Typically 150–400MB per 10-min video.
Final deliverable files
final_spanish.mp4 → 1080p H.264, AAC, burned Spanish subtitles final_hindi.mp4 → 1080p H.264, AAC, burned Hindi subtitles final_arabic.mp4 → 1080p H.264, AAC, burned Arabic subtitles final_japanese.mp4 → 1080p H.264, AAC, burned Japanese subtitles
Audio normalisation is applied automatically — this ensures the cloned voice audio matches broadcast loudness standards and sounds professional on any device.
